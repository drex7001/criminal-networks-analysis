# Spec 07 — UI Evolution

Status: draft (Phase 1 minimal, Phase 2 panels, Phase 4 workspace — v2 numbering, ADR-022) · GOAL.md §29–30

## 1. Strategy

Three stages, no big-bang rewrite:

1. **Phase 1 — keep the explorer.** `app/static/index.html` (Cytoscape.js) unchanged,
   served by aegis-api behind auth, reading the legacy-shaped projection. Zero UI work
   while the foundation moves.
2. **Phase 2 — augment the explorer.** Two additions to the existing page:
   *provenance panel* ("why connected?" — claims, sources, gradings, contradictions
   behind the selected edge/node) and a *review queue* page (plain HTML/htmx or
   minimal JS is fine — function over polish).
3. **Phase 4 — React + TypeScript workspace.** Ontology-driven screens built on the
   Phase 3 generated TypeScript SDK (specs/08); the explorer becomes the graph view
   inside it; legacy page retired at parity.

## 2. Ontology-driven screens (Article XI)

Generic components render from `ui_meta.json` (generated from the ontology):

- **Entity page**: title/subtitle from `display`; claims grouped by predicate; each
  claim row shows assertion type, grading chips (reliability / credibility /
  verification — Article III), validity window, source link, contradiction badge.
- **Claim detail**: full provenance — excerpt, source record, derivative chain down to
  the original file; relations; retraction state.
- **Forms**: create/edit claim drafts built from ontology property/predicate specs —
  adding `vessel` to the YAML yields a working vessel page with no new UI code.

## 4. Interaction principles (binding — GOAL.md §30)

- Search first, expand second; expansion always bounded (hops/types/time/confidence).
- Every claim opens its source; every metric shows its caveat (Article IX).
- Suggested (AI) material is visually distinct from recorded claims — different
  styling, separate legend entry; never mixed silently into the canonical view
  (Article VII).
- Contradictions are a visible lens, not a footnote (Article VIII).
- Current vs historical state clearly separated; as-of mode has a persistent banner.
- No red-means-criminal styling; categories use the ontology palette.
- Restricted data is absent, not teased (no "N hidden" counts — specs/03 §4).
- Retain what already works: layer/category toggles, confidence filter,
  temporal slider with play, cell coloring, source-cited detail cards.
